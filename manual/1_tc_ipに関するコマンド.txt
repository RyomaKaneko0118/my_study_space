ns1、ns2という仮想環境を作る。
ns1-veth0というイーサネットデバイスを作成し、
始点をns1-veth0、終点をns2-veth0とする。
イーサネットデバイスの始点をns1、終点をns2と紐付ける。
ns1側のns1-veth0イーサネットデバイスのエンドポイントを、192.0.2.1/24、
ns2側のns1-veth0イーサネットデバイスのエンドポイントを、192.0.2.2/24、
に設置する。
それぞれのエンドポイントを起動する。

sudo ip netns add ns1
sudo ip netns add ns2
sudo ip link add ns1-veth0 type veth peer name ns2-veth0
ip link show

sudo ip link set ns1-veth0 netns ns1 
sudo ip link set ns2-veth0 netns ns2 

sudo ip netns exec ns1 ip address add 192.0.2.1/24 dev ns1-veth0 
sudo ip netns exec ns2 ip address add 192.0.2.2/24 dev ns2-veth0 

sudo ip netns exec ns1 ip link show ns1-veth0

sudo ip netns exec ns1 ip link set ns1-veth0 up
sudo ip netns exec ns2 ip link set ns2-veth0 up

sudo ip netns exec ns1 ping -c 3 192.0.2.2
sudo ip netns delete ns1
sudo ip netns delete ns2
sudo ip link delete ns1-veth0

ns1、ns2、routerという仮想環境を作る。

ns1-veth0イーサネットデバイスの始点をns1、終点をrouterと紐付ける。
ns1側のns1-veth0イーサネットデバイスのエンドポイントを、192.0.2.1/24、

に設置する。
それぞれのエンドポイントを起動する。

ns2-veth0というイーサネットデバイスを作成し、
始点をns2-veth0、終点をgw-veth1とする。
ns2-veth0イーサネットデバイスの始点をns2、終点をrouterと紐付ける。

sudo ip netns exec ns2 ip link set ns2-veth0 up

sudo ip netns exec ns1 ip address add 192.0.2.1/24 dev ns1-veth0
sudo ip netns exec router ip address add 192.0.2.254/24 dev gw-veth0

ns1、ns2、router1、router2という仮想環境を作る。

ns1-veth0というイーサネットデバイスを作成し、
始点をns1-veth0、終点をgw1-veth0とする。
ns1-veth0イーサネットデバイスの始点をns1、終点をrouter1と紐付ける。
ns1側のns1-veth0イーサネットデバイスのエンドポイントを、192.0.2.1/24、
router1側のns1-veth0イーサネットデバイスのエンドポイントを、192.0.2.254/24、
に設置する。
それぞれのエンドポイントを起動する。

gw1-veth1というイーサネットデバイスを作成し、
始点をgw1-veth1、終点をgw2-veth0とする。
gw1-veth1イーサネットデバイスの始点をrouter1、終点をrouter2と紐付ける。
router1側のgw1-veth1イーサネットデバイスのエンドポイントを、203.0.113.1/24、

gw2-veth1というイーサネットデバイスを作成し、
始点をgw2-veth1、終点をns2-veth0とする。
gw2-veth1イーサネットデバイスの始点をrouter2、終点をns2と紐付ける。
router2側のgw2-veth1イーサネットデバイスのエンドポイントを、198.51.100.254/24、
ns2側のgw2-veth1イーサネットデバイスのエンドポイントを、198.51.100.1/24、
に設置する。
それぞれのエンドポイントを起動する。

ns1のルートテーブルにデフォルトルートとrouter1側のns1-veth0イーサネットデバイスのエンドポイントを設定する。
ns2のルートテーブルにデフォルトルートとrouter2側のgw2-veth1イーサネットデバイスのエンドポイントを設定する。
router1、router2のカーネルにIPv4のルーターとして動作することを設定する。

router1のルートテーブルに198.51.100.0/24とrouter2側のgw1-veth0イーサネットデバイスのエンドポイントを設定する。
router2のルートテーブルに192.0.2.0/24とrouter1側のgw2-veth1イーサネットデバイスのエンドポイントを設定する。

sudo ip link add gw1-veth1 type veth peer name gw2-veth0
sudo ip link add gw2-veth1 type veth peer name ns2-veth0

sudo ip link set ns1-veth0 netns ns1 
sudo ip link set gw1-veth0 netns router1 
sudo ip link set gw1-veth1 netns router1

sudo ip link set gw2-veth1 netns router2
sudo ip link set ns2-veth0 netns ns2

sudo ip netns exec ns1 ip link set ns1-veth0 up
sudo ip netns exec router1 ip link set gw1-veth0 up

sudo ip netns exec router1 ip address add 192.0.2.254/24 dev gw1-veth0

link/etherフィールドはインターフェースに割り当てられたMACアドレスを示す。
この場合は「be:3e:13:08:5b:ea」になる。
inetフィールドは、インターフェースに割り当てられたIPv4アドレスを示す。
この場合は「192.0.2.254/24」になる。
/24はサブネットマスクを示し、この場合は255.255.255.0です。
inet6フィールドは、インターフェースに割り当てられたIPv6アドレスを示しす。
この場合は「fe80::bc3e:13ff:fe08:5Bea/64」になる。
/64はIPv6のサブネットマスクを示し、この場合は64ビット。
stateフィールドが「UP」となっており、インターフェースがアクティブであることを示す。
mtuフィールドは、インターフェースの最大転送ユニット（MTU）を示す。
この場合は1500バイトです。
qlenフィールドは、インターフェースの送信キューの長さを示す。
link-netnsフィールドは、インターフェースがどのネットワーク名前空間にあるかを示す。

Linuxマシンのルーティングテーブルの2つのエントリーを示す。
一つ目のエントリーは、
ホストが宛先IPアドレスに対応するネットワークに直接接続されていない場合に、
どこにパケットを送信するかを示す。
このエントリーは、デフォルトゲートウェイが「192.0.2.254」で、
それに接続されているネットワークインターフェースが「ns1-veth0」であることを示す。
二つ目のエントリーは、
ネットワークインターフェース「ns1-veth0」が192.0.2.0/24ネットワークに属しており、
そのIPアドレスは「192.0.2.1」であることを示す。
