【クライアント証明書の作成】
openssl-client.cnfの設定の確認
cat /etc/pki/tls/openssl-client.cnf

csrと秘密鍵の作成
SSLEAY_CONFIG="-config /etc/pki/tls/openssl-client.cnf" /etc/pki/tls/misc/CA -newreq

Enter PEM pass phrase : （証明書自体のパスワード）
Verifying - Enter PEM pass phrase : （上のパスワードの再入力）
Country Name : JP（そのままEnter）
State or Province Name (full name) : Tokyo
Locality Name (eg, city) : Minato
Organization Name (eg, company) : alice.jp
Organizational Unit Name (eg, section) : （空白のままエンター）
Common Name : alice Client
Email Address : （空白のままエンター or 管理者のメールアドレス）
A challenge password : （空白のままエンター）
An optional company name : （空白のままエンター）

csrをプライベート認証局で署名
SSLEAY_CONFIG="-config /etc/pki/tls/openssl-client.cnf" /etc/pki/tls/misc/CA -sign

Enter pass phrase for /etc/.../CA/private/cakey.pem : プライベート認証局のパスワード
Sign the certificate? [y/n] : （yを入力してエンター）
1 out of 1 certificate requests certified, commit? [y/n] : （yを入力してエンター）

証明書(crt)から配布用PKCS#12形式の証明書を作成
openssl pkcs12 -export -in (証明書のパス) -inkey (秘密鍵のパス) -certfile (CA証明書のパス) -out (出力先のパス)

Enter pass phrase for newkey.pem : 証明書自体のパスワード
Enter Export Password : エクスポートパスワード
Verifying - Enter Export Password : エクスポートパスワード再入力

sftp -i iicom_app_ssh ユーザー名@ホスト名
